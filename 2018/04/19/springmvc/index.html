springmvc

  • 疑难杂症
  • 2018-04-19

最近几天在学习myBatis框架和springmvc框架。因为之前学过SSH框架，所以相对来说比较轻松点，但是昨天还是有个关于springmvc的文件上传和下载有个问题弄得焦头烂额。所以以博客以记之。还有一些关于eclipse的知识记录。

发现问题

;
一直报这个错误，上网搜了很多，发现网上说的主要有两点:第一点是form的mothod要设置成post，第二点需要把表单的enctype设置multipart/form-data。但是我的代码这两点都做到了，还是出现这问题。

分析问题

经过浏览器的地址，和尝试发现了一个问题。我的请求地址名字和网页的名字相同。

 

public class UploadDownloadController {
@RequestMapping(value="/upload")
@ResponseBody
/**
 * 通过multipartFile 来接受文件，如果是多个文件，就使用数组
 * 形参名与表单名一样
 * @param file
 * @return
 */ 


 

<form action="/uploadFile" method="post" enctype="multipart/form-data" >
    用户名：<input type="text" name="username"> <br/>
    头&emsp;像:<input type="file" name="file"/><br/>
    <input type="submit" value="上传">
</form>


这时候提交表单给后台的时候访问的是/upload,但是我打印出name,file值都为空，当时随便打印一个字符，是可以打印出来的，说明是访问到这个方法。但是得不到前台传出来的数据。

解决问题

我把请求地址改成/uploadFile这时就可以解决问题。能够得到数据。具体什么原因，应该是springmvc在源码中可以看出对于一个请求地址，springmvc会生成/地址，例如/upload。会生成/upload，这时你在访问/upload_do，什么的也可以访问到，这时对于一个请求地址和网页地址一样，就会起到冲突。所以会出现上面的问题。

上传文件找不到问题以及解决

我在eclipse中创建了一个upload文件夹，但是发现上传的文件并在这个文件夹中找不到。这时候我打印出来上传的真实路径：
，发现在D:\soft\apache-tomcat-7.0.63\webapps\springmvc_lastly\upload文件夹中。这时我就知道原因了，原来我把网站部署到tomcat的时候doBase写的是默认的路径，即tomcat webapps的文件夹中，这时它会给我生成一个项目，里面的东西都存在那里面。所以这时我只需要右键项目名找到properties,找到resources。然后复制项目的原始路径，替换到doBase中，重新启动服务器就行。例如doBase改成E:\eclipse\springmvc_lastly\WebContent。这时就可以看到eclipse创建的文件中看到了上传的文件了。
